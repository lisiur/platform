## Spacing Scale

Declarative 8-point scale (multiples of 4). All values derived from existing globals.css + shadcn defaults.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps within menu items, inline padding |
| sm | 8px | Tree node horizontal padding, compact element spacing |
| md | 16px | Default element spacing, tree level indentation |
| lg | 24px | Section padding, form field gaps |
| xl | 32px | Layout gaps between role selector and tree |
| 2xl | 48px | Major section breaks (role selector area to tree area) |
| 3xl | 64px | Page-level spacing (top padding) |

**Exceptions:**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| Tree level indent | 16px per level | Matches existing `menu-tree.tsx` line 148: `paddingLeft: level * 16 + 8` |

**Interaction Flow:**

1. Admin lands on `/roles/[roleId]/menus` (or `/roles/menus` with role selector)
2. Admin selects a role from the dropdown вҶ’ tree loads with current assignments pre-checked
3. Admin toggles checkboxes on/off вҶ’ auto-includes children when checking a parent (D-03)
4. Admin clicks "дҝқеӯҳеҲҶй…Қ" вҶ’ PUT request sent, success/error toast shown
5. Changes take effect on next page load (Zustand store re-fetch)

### Checkbox Tree Behavior

- **Check parent вҶ’ all descendants checked** (D-03)
- **Uncheck parent вҶ’ all descendants unchecked** (cascading uncheck)
- **Check child without parent вҶ’ parent remains unchecked** (no auto-include upward)
- **Visual:** Checkbox component from shadcn (`@/components/ui/checkbox`)
- **Tree structure:** Reuse existing `buildTree()` pattern from `menu-tree.tsx`
- **Icons:** Reuse existing `getIcon()` pattern from `menu-tree.tsx`
- **Expand/collapse:** Reuse existing expand/collapse toggle pattern

### Dynamic Sidebar

**Loading flow:**

1. User logs in вҶ’ `useMenuStore.fetchMenus()` called on mount
2. While loading: `Skeleton` components render in sidebar (existing `SidebarMenuSkeleton` pattern)
3. Menus arrive вҶ’ tree built in Zustand store вҶ’ sidebar renders collapsible tree
4. `bottomMenuItems` (profile, settings) remain hardcoded below the dynamic section
